## פריסה ל-suvid.dvirai.com

### מצב פריסה נוכחי — חי על Cloudflare Pages (מ-2026-06-20)

**האתר חי לגמרי על Cloudflare, ללא תלות במחשב הביתי.** ארכיטקטורת serverless חינמית ותמיד-דלוקה:

- **Pages** מגיש את ה-build הסטטי (`dist`). פרויקט: `suvid`, כתובת ברירת מחדל `https://suvid.pages.dev`.
- **דומיין:** `suvid.dvirai.com` הוא custom domain של פרויקט ה-Pages (CNAME ל-`suvid.pages.dev`, proxied). HTTPS אוטומטי.
- **קריאה** (חיפוש/קטלוג/חומר גלם/מחשבון) רצה כולה בלקוח מתוך ה-seed — אין יותר API לקריאה.
- **כתיבה** (דירוגים/הצעות) ב-**Pages Functions** מול **D1** (SQLite serverless). DB: `suvid`,
  מזהה `ef425b33-45c8-4fa8-a22e-bc8762242d6c`, אזור WEUR, binding `DB`.
- Express נשאר רק לפיתוח מקומי ולבדיקות; **לא נפרס**. הלוגיקה משותפת דרך
  `src/domain/ratings.ts` ו-`src/domain/catalog.ts` כך שאין כפילות בין Express ל-Functions.

קבצים רלוונטיים:

- `functions/api/ratings.ts` (POST), `functions/api/ratings/summary.ts` (GET), `functions/api/suggestions.ts` (POST), `functions/_shared.ts`.
- `wrangler.toml` (binding `DB`, `database_id`, `compatibility_date = 2026-05-27`).
- `db/schema.sql`, `db/json-to-sql.mjs` (מיגרציה מ-JSON אם יצטברו דירוגים מקומיים — כרגע ריק).
- סקריפטים ב-`package.json`: `pages:dev`, `pages:deploy`, `d1:schema`, `d1:schema:local`.

נבדק מקצה-לקצה בפרודקשן: home 200, POST דירוג/הצעה → 201, summary מתעדכן, קלט לא תקין → 400,
וקרוס-צ'ק שהוכיח ש-`suvid.dvirai.com` פוגע באותו D1 כמו `suvid.pages.dev`. אומת שהאתר עובד גם
כשהמחשב הביתי כבוי לגמרי.

#### עדכון / deploy מחדש

```powershell
# קוד חדש לאתר:
npm run pages:deploy          # build + wrangler pages deploy

# שינוי סכימת D1 (זהירות — פרודקשן):
npm run d1:schema             # מריץ db/schema.sql על D1 המרוחק

# פיתוח מקומי מול Functions + D1 מקומי:
npm run d1:schema:local
npm run pages:dev
```

הערה על הרשאות: פעולות D1/Pages דורשות `CLOUDFLARE_API_TOKEN` עם הרשאות **D1·Edit** ו-**Pages·Edit**.
הטוקן ששימש לפריסה כולל אותן.

#### מה הוסר (תשתית המחשב הביתי הישנה)

הפריסה הזמנית דרך Cloudflare Tunnel + Task Scheduler **הוסרה** אחרי שהמעבר ל-Pages אומת:

- המשימות `SuvidServer` / `SuvidTunnel` / `SuvidAutoRecover` נמחקו.
- הטאנל `suvid` (`87006c73...`) נמחק. הטאנל `mundial-predictor` לא נגעו בו.
- קבצי `deploy/` (קונפיג טאנל + סקריפטי ensure) נמחקו.

> כל מה שמתחת לכותרת **"רקע ואפשרויות (היסטורי)"** הוא התכנון המקורי של מחשב ביתי + reverse proxy,
> ונשמר רק לתיעוד היסטורי. הוא **לא בשימוש**.

### רקע ואפשרויות (היסטורי)

היעד שסוכם: האפליקציה רצה על המחשב המקומי שמחובר 24/7, והדומיין `suvid.dvirai.com` מפנה אליה.

### אפשרות מומלצת ופשוטה

1. להריץ את Node/Express על פורט פנימי `3001`.
2. לשים Reverse Proxy מקומי שמקבל HTTP/HTTPS מהדומיין ומעביר ל-`127.0.0.1:3001`.
3. להגדיר DNS ל-`suvid.dvirai.com`.
4. להפעיל את השרת כתהליך קבוע.

### DNS

אם למחשב יש IP ציבורי קבוע:

- ליצור רשומת `A`:

```text
suvid.dvirai.com -> YOUR_PUBLIC_IP
```

אם אין IP קבוע:

- להשתמש ב-DDNS או Cloudflare Tunnel.
- Cloudflare Tunnel יכול להיות פתרון נוח אם לא רוצים לפתוח פורטים בראוטר.

### פתיחת פורטים

אם משתמשים ב-Reverse Proxy מקומי:

- לפתוח בראוטר פורט 80 ו-443 אל המחשב.
- לא לחשוף את פורט 3001 ישירות החוצה.

### Reverse Proxy

אפשרויות טובות:

- Caddy - הכי פשוט ל-HTTPS אוטומטי.
- Nginx.
- IIS עם ARR.
- Cloudflare Tunnel, אם רוצים להימנע מפתיחת פורטים.

דוגמת Caddyfile:

```text
suvid.dvirai.com {
  reverse_proxy 127.0.0.1:3001
}
```

### 3. מסך ניהול הצעות מתכון

- כתובת: `https://suvid.dvirai.com/#admin` (גם קישור "ניהול" בפוטר).
- מוגן ב-**מפתח ניהול** (header `x-admin-key`).
- API: `GET /api/suggestions` (רשימה), `PATCH /api/suggestions/:id` (`status`: approved/rejected) — שניהם דורשים מפתח.
  ממומש גם ב-Express (dev/בדיקות) וגם ב-Pages Functions (פרודקשן), עם ולידציה משותפת.
- **המפתח בפרודקשן** מוגדר כ-secret של פרויקט ה-Pages בשם `SUVID_ADMIN_KEY`, ושמור מקומית ב-`.admin-key.txt`
  (gitignored). להחלפת המפתח:

```powershell
'מפתח-חדש' | wrangler pages secret put SUVID_ADMIN_KEY --project-name suvid
# ואז deploy מחדש: npm run pages:deploy
```

- בפיתוח מקומי: קובץ `.dev.vars` עם `SUVID_ADMIN_KEY=...` (gitignored). ב-e2e המפתח הוא `e2e-admin-key`.
- הערה: ההשוואה עושה `trim()` כדי לסבול רווחים/שורות חדשות שנכנסים בטעות לערך ה-secret.

### 4. הצעה מאושרת → אופציה חיה במחשבון

הצעה שאושרה **ויש לה טמפרטורה + זמן** הופכת אוטומטית לחומר גלם "קהילה" חי במחשבון:

- `src/domain/community.ts` — `isPublishable`, `toCommunityIngredient`, `mergeCommunity`.
- API ציבורי `GET /api/community` (Express + `functions/api/community.ts`) מחזיר הצעות מאושרות עם טמפ׳+זמן.
- הלקוח (`src/api.ts` → `loadCommunity`) ממזג אותן ל-seed בזמן ריצה; הן מופיעות בחיפוש ובקטלוג תחת קטגוריה "הצעות קהילה",
  עם תגית "מהקהילה" וביטחון "ניסיון ראשון". אין דירוג לאופציות קהילה (ה-targetId לא בסט המאושר).
- במסך הניהול: הצעה מאושרת עם טמפ׳+זמן מסומנת "חי במחשבון"; בלי טמפ׳/זמן מוצגת הערה שצריך להוסיף.

### 5. נתוני המחשבון מבוססים על כמה מקורות אמינים

המשמש כמקור משלים לרוב האופציות המומלצות לצד Serious Eats / ChefSteps / Anova.

### 6. סדר קטגוריות בקטלוג + ירקות

- סדר הקטגוריות בקטלוג: **בקר, עוף, דגים** ראשונים, ואז כפתור **"עוד…"** שחושף את שאר הקטגוריות
  (`PRIMARY_CATEGORIES` ב-`src/domain/catalog.ts`; הסדר נאכף ב-`buildCatalog`, וקטגוריית "הצעות קהילה" אחרונה).
- קטגוריית טופו: שונתה מ-"צמחי" ל-**"טופו"** (מופיע ישירות).
- קטגוריית **"ירקות"** הורחבה: תפוח אדמה, בטטה, גזר, סלק, אספרגוס, תירס (בנוסף לירקות שורש) — כולל חיפוש מתכוני ירקות.

### 8. הרחבת מאגר נוספת (יוני 2026)

- נוספו **13** חומרי גלם → סה״כ **44** במאגר:
  - בקר: לשון בקר, רוסטביף (שריר/eye of round).
  - עוף: כנפי עוף.
  - דגים: אמנון (מושט), בקלה (cod), פורל (trout).
  - פירות ים: תמנון, לובסטר.
  - חזיר: פילה חזיר, כתף חזיר (פולד פורק).
  - טלה: שוק טלה, כתף טלה.
  - ירקות: כרובית.
- כל אופציה מומלצת מגובה ב-**2+ מקורות אמינים** (URLs אומתו בחיפוש אינטרנט: Anova, Serious Eats,
  ChefSteps, Amazing Food Made Easy, Baldwin). טמפ׳/זמן הותאמו למקור.
- המאגר עכשיו **44** חומרי גלם.

### 9. אימות אמינות נתונים (יוני 2026)

- בוצעה השוואה שיטתית של **כל** טמפרטורה/זמן מומלצים מול מקורות אמינים
  (Baldwin, Serious Eats/Kenji, ChefSteps, Anova, Amazing Food Made Easy).
- רוב הנתונים תאמו במדויק. תוקנו 4 פריטים שהיו בקצה הרך/נמוך של הטווח:
  - **שרימפס** 55→57°C (נקודת המתיקות של Kenji למרקם עסיסי-קפיצי).
  - **חזה ברווז** זמן 1.5→2 שעות (המקורות ממליצים 2-3 שעות).
  - **דניס** 50→52°C (פתיתי ולח במקום שקוף-רך).
  - **לברק** 50→52°C (אותה סיבה). אופציות ה"יציב" עודכנו בהתאם (56/55°C).
- נוסף מבחן רגרסיה `matches verified temperatures/times from reliable sources`
  ב-`src/data/seed.test.ts` שנועל ~30 ערכים מאומתים כדי למנוע סחיפה שקטה.

### 10. חישוב לפי עובי + בטיחות + הוראות צריבה (יוני 2026)

- מודול חדש `src/domain/cooking.ts` (+`cooking.test.ts`):
  - **חישוב זמן לפי עובי** — מודל חימום של Baldwin (זמן ∝ עובי²; `SLAB_COEFF_MIN_PER_MM2 = 0.14`).
    הזמן ה"בסיסי" המאומת נשמר לעובי טיפוסי (`referenceThicknessCm`: 2 ס״מ לדגים/פירות ים, 2.5 אחרת),
    ומתווסף זמן רק לנתח עבה מהרפרנס. בישול ארוך (≥6ש׳, כמו אסאדו) כמעט לא מושפע. תוספת ל"מהקפאה".
  - **בטיחות/פסטור** — `assessSafety(category, temp, time)` מסווג ל-pasteurized/cooked/immediate/raw
    עם תווית והערה. שמרני בכוונה (עוף ≥60°, חזיר ≥58°, בקר/טלה מ-54.5° או בישול ארוך, דגים<50°=סשימי).
  - **הוראות צריבה** — `searGuide(category)` מחזיר זמני צריבה קונקרטיים לכל קטגוריה.
- UI (`src/App.tsx`): קלט **עובי הנתח (ס״מ)** + תיבת **מהקפאה** במקום ה-select הישן; הזמן במחשבון
  ובחישוב "מתי להתחיל" מתעדכן לפי העובי; תג בטיחות צבעוני וטיפ צריבה בכרטיס האופציה.

### 11. טאבים לכיווני הכנה (יוני 2026)

- מודול חדש `src/domain/directions.ts` (+`directions.test.ts`): לכל קטגוריה יש 2 "כיווני הגשה"
  מתוכללים עם שם הנתח (`CATEGORY_DIRECTIONS`), שמתווספים לכיוון ה"חתימה" של כל נתח דרך `withDirections`.
- ה-decorate מופעל ב-`src/api.ts` (`decorate`) על seed + community, כך שכל נתח מקבל **≥3 כיווני הכנה**.
- UI: סקשן "כיווני הכנה" הפך ל**טאבים** (`.direction-tabs`) — כל טאב מציג כיוון אחר (רוטב/תיבול/הגשה).
- e2e: `user can switch between preparation direction tabs`.

### 7. ניווט ו-UX (חזרה אחורה + דסקטופ)

- **ניווט אחורה:** בחירת נתח רושמת כניסה ל-`history` (`pushState` ב-`pushResultHistory`), כך
  שכפתור BACK של הדפדפן/מכשיר מחזיר לקטלוג במקום לצאת מהאתר (`popstate` → `goHome`).
- נוסף כפתור **"→ חזרה לכל חומרי הגלם"** (`.back-bar`) בראש כל מסך תוצאה (נתח / הבהרה / לא נמצא).
- הלוגו **SuVid** הפך לכפתור שמחזיר לדף הבית; כל מעבר גם גולל לראש העמוד.
- בדיקת e2e: `user can navigate back from a cut without leaving the site` (נועלת back-bar + BACK + לוגו).
- **דסקטופ:** מסך הנתח עבר לפריסת **שני טורים** ב-`≥920px` (`.detail-layout`): תוכן מצד אחד,
  סייד־בר פעולה (טיימר "מתי להתחיל" + דירוג + הצעה) מהצד. הקטלוג בדף הבית עובר לרשת דו־טורית ב-`≥720px`.
  רוחב כולל גדל ל-`--maxw: 1080px`. במובייל נשאר טור יחיד ללא רגרסיה.
